# Assets Organization Guide

This document describes how CSS and JavaScript assets are organized across the application.

## CSS Organization

### Main Stylesheets

#### 1. `public/css/style.css`

- **Source**: CoreUI Framework (Bootstrap-based theme)
- **Size**: ~14,970 lines
- **Usage**: Main framework styles loaded in all pages
- **Location in Layout**: `resources/views/components/layout.blade.php`

#### 2. `public/css/custom-premium.css`

- **Purpose**: Custom application styles
- **Size**: ~50 lines
- **Sections**:
    - **Settings Page Styles**:
        - `.settings-rate-option` - Form layout styles
        - `.form-control` & `.form-select` - White text color for form inputs
    - **Placeholder Styles**: Semi-transparent white for disabled fields
- **Blade Files Refactored**:
    - `resources/views/settings/index.blade.php` (removed inline `<style>` block)
- **Location in Layout**: `resources/views/components/layout.blade.php`

#### 3. `public/css/examples.css`

- **Purpose**: Example/demo styles from CoreUI
- **Status**: Can be removed in production

#### 4. `public/css/invoice.css`

- **Purpose**: Styles for PDF invoice generation
- **Usage**: Loaded in `resources/views/maintenance_bills/invoice_pdf.blade.php`

### CSS Location in Layout

All main stylesheets are loaded in:

```html
<!-- resources/views/components/layout.blade.php -->
<link href="{{ asset('css/style.css') }}" rel="stylesheet" />
<link href="{{ asset('css/custom-premium.css') }}" rel="stylesheet" />
<link href="{{ asset('css/examples.css') }}" rel="stylesheet" />
@stack('styles')
<!-- For inline blade styles -->
```

## JavaScript Organization

### Main Scripts

#### 1. `public/js/script.js`

- **Size**: ~3,707 lines
- **Status**: Contains migrated Blade scripts
- **Key Functions**:
    - Settings page logic (penalty/discount toggles)
    - Currency formatting utilities
    - CSRF token handling
    - JSON data parsing
    - Data table initialization
    - Form validation
    - API request handling
- **Location in Layout**: `resources/views/components/layout.blade.php`

#### 2. Dynamic/Inline Scripts in Blade Files

These scripts contain Blade template syntax and MUST remain in their respective blade files:

##### `resources/views/settings/index.blade.php`

- **Location**: `@push('scripts')` at end of file
- **Contains**:
    - `window.SMP_SETTINGS_CONFIG` - Configuration object with:
        - Available currencies
        - Currency symbol
        - Society GPS coordinates
        - CSRF token
        - API routes
    - Event listener for debugger toggle
- **Why**: Contains dynamic Blade variables like `@json()`, `{{ }}`, `{{ csrf_token() }}`

### Blade Files with DataTable Scripts

The following blade files load DataTable scripts dynamically:

- `resources/views/name_transfer_bills/index.blade.php`
- `resources/views/expenses/index.blade.php`
- `resources/views/flat_documents/index.blade.php`
- `resources/views/users/index.blade.php`
- `resources/views/maintenance_bills/index.blade.php`
- `resources/views/flats/index.blade.php`
- `resources/views/blocks/index.blade.php`
- `resources/views/flat_types/index.blade.php`
- `resources/views/expense_categories/index.blade.php`
- `resources/views/complains/index.blade.php`
- `resources/views/residents/index.blade.php`

All use: `{{ $dataTable->scripts(attributes: ['type' => 'module']) }}`

**Note**: These scripts are auto-generated by Yajra DataTables library and cannot be moved to static files.

### JavaScript Stack Points in Layout

```html
<!-- resources/views/components/layout.blade.php -->
<script src="{{ asset('js/script.js') }}"></script>
@stack('scripts')
<!-- For inline blade scripts -->
```

## Best Practices

1. **CSS Updates**:
    - Add custom application styles to `public/css/custom-premium.css`
    - Keep framework styles in `public/css/style.css`

2. **JavaScript Updates**:
    - Add reusable JavaScript functions to `public/js/script.js`
    - Keep Blade-dependent scripts in blade files with `@push('scripts')`
    - Use `{{ $dataTable->scripts() }}` for DataTable initialization

3. **Asset Versioning**:
    - Main assets include cache-busting: `?v={{ filemtime(...) }}`
    - Ensure updated assets are loaded by clearing browser cache if needed

4. **Performance**:
    - All CSS is loaded in `<head>` for critical rendering path
    - JavaScript is loaded at end of `<body>` for non-blocking loading
    - External CDN resources are used for third-party libraries
